About & Help

kobo2readwise gets highlights out of your sideloaded Kobo books and into Readwise.

What it is

When you sideload a book onto a Kobo — an EPUB or PDF added through Calibre, email, or a USB cable — the highlights you make in it are saved only on the device, in a local database file called KoboReader.sqlite. They never reach Kobo's cloud, so Readwise's automatic Kobo sync can't see them.

kobo2readwise bridges that gap. It reads KoboReader.sqlite right in your browser and sends those highlights to your Readwise account. No app to install, nothing to configure on the device.

What's in scope (and what isn't)

In scope: highlights from books you put on the Kobo yourself (sideloaded EPUB/PDF).

Not needed here: books from the Kobo store, Kobo Plus, or library loans (OverDrive). Those already sync to Readwise through Readwise's own Kobo integration — kobo2readwise deliberately leaves them alone (and offers a "sideloaded only" filter so you don't create duplicates).

How it works

  • Your browser opens KoboReader.sqlite locally using the File System Access API — the file is never uploaded.
  • It's parsed in-page with sql.js (SQLite compiled to WebAssembly).
  • The highlights are formatted for Readwise and posted through a thin server-side proxy — needed only because the Readwise API can't be called directly from a browser.

Your data & your token

The proxy exists for one reason — to relay the request to Readwise — and it is built to forget:

  • Your Readwise token is forwarded to Readwise and never stored, cached, or logged.
  • There is no database and no request-body logging on the server.
  • Your Kobo database and highlights are processed in your browser; the file itself never leaves your computer.
  • The whole thing is open source — you can read exactly what the proxy does.

Requirements

The File System Access API only exists in a couple of desktop browsers, so kobo2readwise needs one of:

Chrome — desktop Edge — desktop

Firefox, Safari, and phone/tablet browsers can't run it. This is a hard browser limitation, not a choice we can work around.

How to use it

  1. Connect your Kobo to your computer with a USB cable and let it mount as a drive.
  2. On that drive, open the hidden .kobo folder and find KoboReader.sqlite. (Tip: if you don't see hidden folders, enable "show hidden files" in your file manager. You can also copy the file to your Desktop first.)
  3. Back on the app, click Choose KoboReader.sqlite… and select that file.
  4. Review the highlights it found. Keep Only sideloaded books checked unless you know you want everything.
  5. Paste your Readwise access token from readwise.io/access_token and click Sync to Readwise.

Troubleshooting

  • "No highlights found" — the database has no text highlights, or they're all from store/library books while the sideloaded filter is on. Try unchecking the filter.
  • "Readwise rejected the token" — double-check you copied the full token from the access-token page.
  • The library couldn't load — an ad blocker or restrictive network may be blocking the CDN that hosts sql.js; allow cdn.jsdelivr.net and retry.
